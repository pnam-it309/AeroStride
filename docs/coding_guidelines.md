# Quy định và Cách làm việc (Coding Guidelines)

Tài liệu này quy định các tiêu chuẩn và cách làm việc chung cho toàn bộ dự án AeroStride.

## 1. Quy định về Comment code
- **Bắt buộc:** Sau khi làm xong 1 module hoặc viết một module/hàm mới, **phải thêm comment vào trước đoạn code** để giải thích rõ đoạn code đó làm chức năng gì, luồng xử lý ra sao.
- Mục đích: Giúp các thành viên trong team dễ dàng đọc hiểu, bảo trì và phát triển tiếp dựa trên các module đã có.

## 2. Quy chuẩn chung
- Viết code sạch (clean code), rõ ràng, tên biến/hàm mang ý nghĩa cụ thể.
- Tuân thủ cấu trúc thư mục và kiến trúc chung của dự án ở BE và FE.
